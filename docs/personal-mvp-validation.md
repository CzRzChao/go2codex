## Safety decisions

- Production Finder preference mutation remains disabled. No public atomic serialization boundary could prevent a concurrent Finder or `cfprefsd` update from being overwritten, so setup and removal use explicit Finder Command-drag interactions.
- Finder status reads synchronize the public preferences cache and require live, disk, and second-live snapshots to agree. No Finder preference setter or Finder restart is connected.
- A verified Installed state may write and read back a versioned receipt only in Go2Codex's own preferences. Invalid historical receipt data cannot override independent current URL, AliasRecord, and signed-identity proof.
- Release manual setup is offered only from `/Applications` or the current user's Applications folder. Debug keeps its documented development-location exemption.
- Apple Event sending code is confined to the Launcher, while macOS attributes the nested invocation's TCC responsibility to the outer Go2Codex application. Both bundles therefore carry the entitlement and localized usage text; ordinary Settings launches still send no Apple Events.
- The frozen pre-SOP build is ad-hoc signed. The strict local SOP migrates it once to Apple Development and then requires stable Team and designated requirements for later Personal updates; Public Release later uses Developer ID and notarization.
- Debug smoke is fail-closed: a pass receipt is invalid while any pending or staging file exists. Release overlay state is also fail-closed and owner-bound; install and rollback cannot recover each other's transaction, and unknown or conflicting evidence is preserved without changing the installed app.
- A real Finder toolbar Option-click closed the invoking viewer window. Querying Finder after that system gesture could resolve a different background window, so Shift-click is the only enabled Alternate Trigger, Disabled is the only alternative, and Option-click is unsupported and must not be used. Finder/Launch Services `oapp` and `rapp` have no documented modifier value bound to the click: when the Launcher's startup snapshot still observes Option or Option+Shift, it fails with `finder-option-modifier-unsupported` before any Finder query, but rapid release cannot be classified strictly. The modifier safety gate remains open, with no retry, fallback, global key or modifier monitor, or Accessibility path. The picker panel's temporary mouse-down-only monitor is limited to outside-click cancellation while visible.
- A real Shift-click displayed the correct four-item picker but it vanished immediately, and the strong-activation-only repair also flashed. The following hard-foreground readiness build passed its first real Shift check but failed the second with `target-picker-activation-timeout`; the installed mouse-release-plus-best-effort-activation `NSMenu` repair also flashed. The frozen installed baseline's picker waits up to one second only for physical mouse release, then uses a compact borderless `.nonactivatingPanel` with `orderFrontRegardless()` and `runModal` rather than activating Go2Codex or cancelling on focus loss. While visible it installs a temporary global mouse-down monitor for outside clicks and removes it immediately when one-shot selection, Escape, or cancellation finishes. Historical Finder checks confirmed stable display, silent Escape/outside cancellation, one exact Codex App Handoff, and five consecutive open/Escape cycles. A separate rapid-overlap click check remains pending.
- Finder status `-1728` does not prove that no viewer window exists; it reports an unavailable Apple Event object. Go2Codex maps it to `finder-object-unavailable`, presents “The original Finder folder is no longer available,” and performs no retry or folder substitution.
- Terminal.app has no verified Apple Event for creating a new tab. With an existing Terminal window and New Tab selected, Go2Codex fails explicitly before submitting a command. Current source repairs cold New Window and no-window New Tab by opening Terminal's exact application URL with the command as its one initial Apple Event, and retries that open path once only after a direct `-600` process-exit race. For iTerm2, the live bare-`type` result invalidated Swift-owned create/write sequencing; current source keeps the declared tab/window creation and exact-session write inside one precompiled parameterized AppleScript handler. The user has verified the running-with-window and running-without-window states; the fully quit state exposed an extra default window because the handler sentinel arrived too late. Current source now sends that sentinel as the command-free `NSWorkspace` initial event before placement work, while retaining the unchanged compiled handler. Updated Installed Debug cold-start validation remains pending.

## Exit-gate exceptions

The original M6 gate requiring Terminal.app New Tab support is not met because no safe creation event was verified. The original M8 automatic Install, Repair, and Uninstall gate is not met because the required cross-writer serialization boundary is unavailable. Both paths fail closed and are documented rather than being reported as successful.

The modifier click-bound safety gate is also open. Finder/Launch Services do not provide the originating toolbar click's modifier state through `oapp` or `rapp`, and the Personal MVP intentionally adds no global key or modifier monitor or Accessibility permission. The picker panel's temporary mouse-down-only global monitor detects outside clicks only while that panel is visible and does not recover the originating click's modifiers. Holding Shift until the picker is stable is the current operating constraint; Option-click is unsupported, and rapid release cannot be guaranteed to take the intended branch. The Personal MVP cannot be marked fully complete against the original M0–M9 definition until these product-gate exceptions and the modifier safety gap are explicitly resolved or accepted.
